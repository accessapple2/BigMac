## D1 — Dual-queue OllamaQueue refactor

**Problem:** Single global `OllamaQueue` serializes ALL Ollama calls system-wide (bigmac + Ollie GPU). When one host is slow (model swap, cold load), it blocks requests destined for the OTHER host, even though they're independent machines.

**Today's mitigation:** Moved all non-phi3 agents off bigmac → Ollie, so bigmac queue only runs phi3:mini (no model swaps). Eliminates the problem for current fleet composition.

**Proposed fix:** Split `OllamaQueue` into per-host instances. `OllamaProvider` selects its queue based on URL at init time. Each host drains independently in parallel.

**Estimated size:** ~50 LOC in `engine/ollama_queue.py` + `engine/providers/ollama_provider.py`

**Priority:** Low — today's routing move fully mitigates for current fleet

**Revisit trigger:** New host added (e.g. bigmac M4 Pro upgrade, third inference box), OR bigmac queue gets a non-phi3 resident again, OR queue depth metrics show stalls > 2x per day
